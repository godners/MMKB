# 如何为 MMKB 贡献

感谢你对 **MMKB - 个人知识库** 的关注！

本仓库主要记录相关知识笔记，采用 [CC BY-NC-SA 4.0](LICENSE) 许可协议。

## 欢迎的贡献类型

- 修正现有笔记中的错误（事实错误、笔误、格式问题）
- 补充新的药物、物质或机制的知识
- 改进现有内容的结构、清晰度和可读性
- 翻译或增加中英双语对照
- 优化仓库文档、脚本或自动化流程

## 如何贡献

### 1. 提交 Issue

如果你发现错误或有好的建议，请先创建一个 Issue，描述清楚：

- 问题所在的文件路径
- 具体错误或建议内容
- （可选）你的修改思路

### 2. 提交 Pull Request (PR)

1. Fork 本仓库
2. 创建一个新的分支（建议使用 `feat/药物名称` 或 `fix/文件名` 命名）
3. 在对应目录下修改或新增 Markdown 文件
4. 确保文件格式清晰，使用统一的模板（如果有）
5. 提交 Pull Request 并描述修改内容

### 贡献规范

- **文件名**：使用中文，尽量简洁有意义，避免特殊符号（如 `青霉素钠.md` 而不是 `青霉素钠,.md`）
- **内容风格**：客观、专业、基于可靠来源。个人经验请注明“个人笔记”或“经验分享”
- **格式**：使用标准的 Markdown 语法，标题层级清晰
- **图片**：如需添加图片，请放在 `assets/` 文件夹中，并使用相对路径
- **许可协议**：提交的内容默认遵循本仓库的 [CC BY-NC-SA 4.0](LICENSE) 许可

## 不接受的贡献

- 纯广告或推广内容
- 违反 CC BY-NC-SA 许可协议的内容
- 大量低质量或机器生成的内容

## 联系方式

- Issue 是最推荐的沟通方式
- 你也可以通过 GitHub 直接联系仓库作者
