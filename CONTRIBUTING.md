# 如何为 MMKB 贡献

感谢你对 **MMKB - 个人知识库** 的关注！本仓库主要记录医学、药理、生理、中医及相关机制的 Markdown 笔记，采用 [CC BY-NC-SA 4.0](LICENSE) 许可协议。

## 欢迎的贡献类型

- 修正现有笔记中的事实错误、笔误、格式或链接问题
- 补充新的药物、物质、机制或临床观察记录
- 改进笔记结构、可读性、表格或图示
- 添加中英双语对照、参考文献或更新日期
- 优化仓库文档、自动化脚本（GitHub Actions）、模板或导航
- 提出新目录结构或标签系统的建议

## 如何贡献

### 1. 提交 Issue（推荐先走此步）

发现错误、缺失内容或有改进想法时，请先创建 Issue，并清晰描述：

- 问题所在的文件路径（如 `常用药物记录/烟酰胺（VB3）.md`）
- 具体问题或建议内容
- （可选）你的修改思路或参考来源

### 2. 提交 Pull Request (PR)

1. Fork 本仓库
2. 创建新分支（推荐命名规范）：
   - 新增笔记：`feat/药物名称` 或 `feat/机制名称`（例如 `feat/烟酰胺补充机制`）
   - 修复问题：`fix/文件名`（例如 `fix/烟酰胺（VB3）.md`）
   - 文档改进：`docs/改进内容`（例如 `docs/更新Markdown规则`）
3. 在对应目录下修改或新增 Markdown 文件
4. **强烈建议**在文件开头添加统一的 YAML frontmatter（见下方模板）
5. 提交 PR 时，请在描述中填写以下模板：

   **PR 描述模板**：

```markdown
# 变更类型
 - [ ] 修复错误
 - [ ] 新增内容
 - [ ] 改进结构/格式
 - [ ] 其他
# 修改内容
（简要说明改动了哪些文件和具体变更）
# 参考来源
（列出可靠来源，如 PubMed、文献、临床指南等）
# 测试/验证
（如有，说明如何验证内容的准确性）
```

1. 提交后，等待 Review。作者会尽快回复。

## 贡献规范

- **文件名**：使用中文 + 必要英文缩写，简洁有意义（例如 `烟酰胺（VB3）.md`），避免特殊符号和空格。
- **内容风格**：客观、专业、以可靠来源为基础。个人经验请明确标注“个人观察”或“经验分享”。敏感内容（如药物使用）需添加免责声明。
- **格式要求**：
- 使用标准的 Markdown 语法，标题层级清晰。
- 表格对齐、代码块正确。
- 图片请放入 `assets/` 目录，并使用相对路径。
- **推荐在文件最顶部添加 YAML frontmatter**（见下方示例）。
- **许可协议**：所有提交内容默认遵循仓库的 [CC BY-NC-SA 4.0](LICENSE) 许可。

### 推荐的 YAML Frontmatter 模板（放在每篇笔记最顶部）

```yaml
---
title: 笔记标题（建议与文件名一致）
last_updated: 2026-04-19
tags: [药物, 机制, 皮肤, 激素]   # 多个标签用数组
categories: [常用药物记录, 皮肤养护资料]
aliases: [VB3, Nicotinamide]     # 别名，便于搜索
sources: 
- "PubMed: PMID XXXXX"
- "临床指南名称"
disclaimer: 本笔记为个人知识整理，仅供参考，不构成医疗建议。
---
```

### 不接受的贡献

- 纯广告、推广或商业内容
- 违反 CC BY-NC-SA 许可协议的内容
- 大量低质量、未经验证或纯机器生成的内容
- 未经标注的个人医疗建议（可能引发误导）

### 联系方式

- 可直接在 PR 中 @GodnersRen（即仓库作者）
- 欢迎讨论知识库的长期方向（如标签系统、知识图谱）
