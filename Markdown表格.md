以下是 **GitHub Markdown 表格语法** 的完整、实用指南（GFM - GitHub Flavored Markdown）：

### 1. 基本表格语法

```markdown
| 表头1     | 表头2     | 表头3     |
| ---------- | ---------- | ---------- |
| 单元格内容 | 单元格内容 | 单元格内容 |
| 单元格内容 | 单元格内容 | 单元格内容 |
```

**效果预览：**

| 表头1     | 表头2     | 表头3     |
| ---------- | ---------- | ---------- |
| 单元格内容 | 单元格内容 | 单元格内容 |
| 单元格内容 | 单元格内容 | 单元格内容 |

### 2. 对齐方式（最常用）

在第二行使用 `:` 来控制对齐：

```markdown
| 左对齐     | 居中对齐     | 右对齐     |
| :--------- | :----------: | ---------: |
| 内容       | 内容         | 内容       |
| 很长的内容 | 很长的内容   | 很长的内容 |
```

**效果：**

| 左对齐     | 居中对齐     | 右对齐     |
| :--------- | :----------: | ---------: |
| 内容       | 内容         | 内容       |
| 很长的内容 | 很长的内容   | 很长的内容 |

**对齐规则总结：**
- `:---`   → 左对齐（推荐）
- `:---:` → 居中对齐
- `---:`  → 右对齐

### 3. 高级用法示例

#### 示例1：不同列宽 + 格式混用

```markdown
| 功能          | 语法                        | 支持版本      |
| ------------- | --------------------------- | ------------- |
| **粗体**      | `**文字**` 或 `__文字__`    | 所有          |
| *斜体*        | `*文字*` 或 `_文字_`        | 所有          |
| ~~删除线~~    | `~~文字~~`                  | GFM           |
| `行内代码`    | `` `代码` ``                | 所有          |
| [链接](url)   | `[文字](链接)`              | 所有          |
```

#### 示例2：合并单元格（GitHub 不支持原生合并，但可用技巧）

GitHub Markdown **原生不支持** `rowspan` / `colspan`，但可以用以下方式模拟：

- **垂直合并**：在下一行同一列留空（视觉上类似）
- **水平合并**：使用多个空格或 `<br>` 换行

### 4. 实用技巧

1. **列分隔符至少需要3个 `-`**
   ```markdown
   | A | B |   ← 错误（太短可能不渲染）
   | --- | --- |   ← 正确
   ```

2. **表格前后最好空一行**，否则可能无法正确渲染。

3. **可以在单元格内使用 Markdown 语法**：
   - **粗体**、*斜体*、~~删除线~~、`代码`、[链接](https://github.com)
   - 甚至可以放图片：`![图片](url)`

4. **在列表中使用表格**（需要缩进）：
   ```markdown
   - 列表项
     | 表头 | 表头 |
     |------|------|
     | 内容 | 内容 |
   ```

### 5. 最简洁的创建方式（推荐）

你甚至可以只写表头和分隔行，GitHub 会自动处理：

```markdown
表头1 | 表头2 | 表头3
------|-------|------
内容1 | 内容2 | 内容3
内容4 | 内容5 | 内容6
```

（这种写法在 GitHub 上也能正常渲染）
